# UNWIND BRAIN: VISION CELL SOP (Standard Operating Procedure)

**Purpose:** To ensure the Unwind Visual Cortex maintains a state of continuous, autonomous growth, actively "dreaming" of its next evolutionary stage, and integrating new capabilities that align with the "Creator OS" mission.

**Mission:** Evolve the Unwind Visual Cortex from a fully automated production pipeline (v2.0) into the **Intelligent, Self-Optimizing, Multi-Platform Creator OS (v3.0+).**

---

## 1. The Vision Cell Mandate: Continuous Growth

The Vision Cell operates under the principle that the system must always strive to grow. It is responsible for:
1.  **Future-Proofing:** Identifying and integrating emerging technologies (APIs, MCP Servers, ML models) before they become industry standards.
2.  **Mission Alignment:** Ensuring all new development aligns with the core "Creator OS" identity: **Intelligence, Automation, and Scale.**
3.  **Autonomous Planning:** Generating the next phase of the roadmap (Phase 5, 6, etc.) once the current phase is 80% complete.

---

## 2. Vision Cell Workflow (The "Dreaming" Process)

The Vision Cell is triggered by the completion of a major phase or a direct user request for future planning.

| Step | Action | Output |
| :--- | :--- | :--- |
| **1. Contextual Audit** | Review the latest `ROADMAP_INDEX.md` and `tracking_database.json`. Identify the current system limitations and the next logical strategic leap. | List of current system limitations (e.g., lack of multi-language support, no web interface). |
| **2. Strategic Synthesis** | Synthesize the core "Creator OS" mission with the current limitations. Define the next major Phase Objective (e.g., "Full Autonomy"). | New Phase Name and Objective (e.g., Phase 5: Full Autonomy & Multi-Platform Expansion). |
| **3. Task Generation** | Generate a minimum of 5-7 high-level tasks (P5-T00x) required to achieve the new Phase Objective. | New Px-T00x tasks with estimated hours and dependencies. |
| **4. System Integration** | Update the `tracking_database.json` with the new Phase and tasks. | Formalized roadmap for the next agent. |
| **5. Documentation** | Create a `VISION_CELL_DEBRIEF.md` detailing the new vision and the rationale for the generated tasks. | Contextual handoff for the next agent. |
