## Key Learnings & Insights

1.  **Python-UXP Integration Requires a Bridge:** The most critical finding is that direct Python-to-UXP communication is not possible. A communication layer, such as a WebSocket server, is mandatory. This fundamentally shapes the architecture of the integration.
2.  **The `morim3/mcp_adobe_premiere` Project is a Key Reference:** This open-source project provides a working proof-of-concept for the required WebSocket architecture, significantly de-risking the implementation phase.
3.  **UXP API is Asynchronous:** All method calls in the Premiere DOM API are asynchronous and must be handled with `await`. This is a major departure from the older synchronous ExtendScript model and requires careful programming to manage promises.
4.  **Data Integrity is Paramount:** The initial failure to find a task highlights the absolute dependency on the tracking database's accuracy. The agent's ability to detect and repair this is a key success factor for the multi-agent system.
