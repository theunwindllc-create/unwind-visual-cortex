## Key Learnings & Insights

1. **Pexels API is Well-Designed:** Clean REST API with good documentation. Easy to integrate and reliable.

2. **Rate Limiting is Critical:** With 200 req/hour limit, a single video project with 20 B-roll shots would use 20 requests. Conservative limiting (180/hour) provides safety margin.

3. **Streaming Downloads are Essential:** Large HD video files require streaming downloads with progress indicators for good UX.

4. **Error Handling Must Be Comprehensive:** Network issues, API errors, and rate limits all need specific handling to prevent crashes.

5. **Convenience Functions Improve Adoption:** The `search_and_download_broll()` function makes integration trivial - one line of code to get B-roll.

### Performance Considerations

With 180 requests/hour conservative limit:
- **10 B-roll shots:** 10 requests = ~3 minutes
- **20 B-roll shots:** 20 requests = ~7 minutes
- **50 B-roll shots:** 50 requests = ~17 minutes

This is acceptable for the use case. Most videos will have 5-15 B-roll shots.

### Future Optimization Opportunities

1. **Batch Processing:** Search for multiple shots in parallel
2. **Caching:** Store search results to avoid re-searching
3. **CLIP Integration:** Use AI to match descriptions to videos more accurately
4. **Pixabay Fallback:** If Pexels has no results, try Pixabay API
5. **Metadata Extraction:** Analyze video content (colors, subjects, mood) for better matching
