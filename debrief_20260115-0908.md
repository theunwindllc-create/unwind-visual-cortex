## Challenges Encountered

### Challenge 1: Understanding the Premiere UXP Markers API
**Description:** The initial implementation of the `addMarkers` function was incorrect because it was attempting to use a static method (`createAddMarkerAction`) that doesn't exist. The correct pattern requires getting a `Markers` instance for a specific sequence first.  
**Resolution:** ✅ This was resolved by carefully reading the official Adobe UXP documentation, which clarified that `createAddMarkerAction` is an instance method. The research was documented in `marker_api_research.md`, and the code in `premiere_api.ts` was updated accordingly.  
**Impact:** This reinforces the importance of thorough documentation review and establishes the correct pattern for all future marker-related development.

---

## Key Learnings & Insights

1.  **Premiere UXP Markers API Pattern:** The correct way to add markers is to first get a `Markers` instance for a specific sequence using the static `Markers.getMarkers()` method, and then call `createAddMarkerAction()` on that instance.
2.  **Strategic Marker Placement:** A simple but effective algorithm can be used to place CTA markers at strategic points in a video based on duration and engagement goals. This can be a powerful tool for automating video optimization.
3.  **Importance of Type Definitions:** Enhancing the `Marker` interface in `types.d.ts` to include CTA-specific metadata makes the code more readable, maintainable, and less prone to errors.
